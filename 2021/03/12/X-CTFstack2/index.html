数组越界

这里主要学习一下数组越界，最近刷到攻防世界的题目，然后发现一个数组越界的题目，但是却不知道该如何利用和确定偏移。

然后利用2019-4月的全国的信安竞赛you_pwn学习了一下数组越界

  • you_pwn
  • stack2

先来看you_pwn

Arch:     amd64-64-little
RELRO:    Partial RELRO

PIE:      PIE enabled

首先明确数组越界的发生是，边界检测不严格而发生的，数组向高地址生长，所以可以覆盖ebp和ret等。

危险函数在这里，对输入的v1没有边界检测，导致数组越界。

偏移比较简单0x150+0x8

然后泄露出返回地址，应为开启了pie，所以需要获得栈基址。

补充一个知识点，基址的后三位都是0

后来一直不对，看了师傅的wp才发现，这个函数不行，我也不知道为啥，所以我们需要main函数返回地址的偏移。

我在sub_B53函数的起始位置下了一个断点，然后运行到变量初始化完毕，查看了一下栈的结构。

计算出偏移，(数组从esp-0x10开始，一直到esp-0x288)所以偏移是0x278

然后，泄露基址，泄露libc，ROP得到shell

代码较长，但是逻辑不是很复杂。最后的v5没有检验，导致越界。

但是偏移不可以直接算，因为这个有点非常规，这里借助chuj师傅的博客解释一下。

所以我们要找到数组的首地址，才可以确定偏移。仔细观察一下，数组赋值的时候。

for ( i = 0; i < v5 && (signed int)i <= 99; ++i )
{
  __isoc99_scanf("%d", &v7);
  v13[i] = v7;
}

查看一下相应的汇编代码。

.text:080486B8                 push    offset aD       ; "%d"
.text:080486BD                 call    ___isoc99_scanf
.text:080486C2                 add     esp, 10h
.text:080486C5                 mov     eax, [ebp+var_88]
.text:080486CB                 mov     ecx, eax
.text:080486CD                 lea     edx, [ebp+var_70]
.text:080486D0                 mov     eax, [ebp+var_7C]
.text:080486D3                 add     eax, edx
.text:080486D5                 mov     [eax], cl

基本上可以确定，赋值发生在，mov ecx,eax和mov [eax],cl且cl是v7。

所以这时候eax就是首地址，（第一次赋值）

下个断点看一下此时eax的值，0xffffce18然后下断点给ret就可以看到返回地址的地址。

0xffffce9c得到偏移，0x84然后把ret地址改成后面的地址就可以了（程序给了后面）

总结

两个题目，都有点非常规的地方，我们每次利用数组越界都只能更改一个字节所以需要循环重复修改，第二问偏移寻找利用了一点技巧
