好久没有写题目了，最近被数据结构压的。。。今天有时间，猛刷了几道题。这一到也算，比较有意思的题目，（我认为）

记录一下。

题目有点奇怪。

int vulnerable()
{
  puts("HaHaHa!\nWhat else can you do???");
  close(1);
  close(2);
  return shell();
}

前几天复现8bit的时候，刚好学了一下close这个东西。close(1)和close(2)实际上是关闭了两个流。

fd是文件描述符

fd = 0 表示标准输入流

fd = 1 表示标准输出流

fd = 2 表示标准错误流

这里一来就关闭了输出和错误流，只允许我们输入。所以需要重新打开输出流。然后找了一下资料，找到这样一个东西。

看这个链接

exec 1>&0 $ exec 2&0就完成了输出输出流的重定向。就愉快的拿到了shell

解释，把1，2重定向到输入流。但是看了他的解释还是没懂。在懂的路上学了几个小东西

  • /dev目录
  • fifo管道
  • exec指令

首先 /dev 包含了linux中使用的所有外部设备，由于linux会把所有的东西当作文件，所以这些设备都有相应的文件，即相应的端口。

fifo是先进先出的缩写，和pipe管道有很大的联系。pipe是程序创建的进程之间通信的管道。fifo是一种命令管道

了解深入看这里

exec有两个作用

  1. 代替shell执行命令，区别是shell执行完之后会回到shell，而exec会直接退出。
  2. 文件重定向，也就是exec 1>&0这样将文件描述符为1的文件重定向到0上
