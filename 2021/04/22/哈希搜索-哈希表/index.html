概述

首先，明确哈希的概念。在我看来，hash就是一种映射函数，这种函数会把我们的数据转化成关键值，然后关键值映射为地址，最后存入hash表里面。这样做的好处是，当我们内存足够大，或者哈希函数选择的够好时，查找所消耗的时间复杂度几乎为1。然而，哈希算法的选择和表的建立方法，一直是难事。下面给出几种常见的哈希算法和链式地址法的哈希表建立

除模取余法

这应该是最简单的一种哈希算法了。核心是，模数的选取。（这里如果发生冲突，直接舍弃，方便后续计算冲突率）

当我们哈希表的长度为n时，选取的模数应该是小于n的最大素数，这样做可以最大程度的减少冲突率。

实现如下。

int prim[50000];
int primNum=0;
int primLessN(int n)
{
  prim[0]=2; //2是最小的素数
  primNum++; 
  for(int i=3;i<n;i+=2){
    bool isPrim=1; //isPrim用来判断一个数是否为素数
    for(int j=0;j<primNum;j++){
      if(i%prim[j]==0){
        isPrim=0;
        break; //没加break之前， 当n=10 0000时，times=2 5239 6936次 （2.5亿） ，加了之后times=4626 4478次 （4.5千万次） 
      }
       
    } 
    if(isPrim) prim[primNum++]=i;//如果是素数，则存入prim素数数组 
  } 
  return prim[primNum-1];
}

随机乘数法

随机乘数法的原理是

  • 随机获取一个0-1的小数k
  • 把每个值成这个小数，例如 n*k,获得n*k的小数部分 kk
  • 最后，计算kk*n 然后取整数部分，作为关键值

同样的是为了避免大量的冲突率。代码实现如下，这里用了别的方法来生成随机小数。同样的，这里减少了实验的样本大小

太大会造成运行时间过长。这里取MAXSIZE = 100000

double Getlittle(double n, double m)
{
    default_random_engine e(time(0));
	uniform_real_distribution<double> u(0.0,1.0);
	return u(e);
}

double seed = Getlittle(0.0,1.0);
int randKey(int n)
{
    double m = seed * n;
    while(m>1)
    {
        m = m-1;
    }
    m *= n;
    int cont = 0;
    while(m>1)
    {
        cont++;
        m = m-1;
    }
    return cont;
}

基数转化法

这种方法也比较简单，原理是，把原来十进制的数当成别的进制（我选择的是13）然后再转化为10进制

int SwitchHash(int key)
{
    int count = 1;int num = 10;
    while(key/num>0)
    {
        count++;
        num = num*10;
    }
    int result = 0;
    for(int i=0;i<count;i++)
    {
        num = (key/pow(10,i));
        num = num%10;
        result += num*pow(13,i);
    }
    int final = result%10 + ((result/10)%10)*10 + ((result/100)%10)*100;
    return final;
}

这种就比较迷，可能是实现有问题，冲突率随着数量的增加也增加,100000的时候大概是0.9，10的时候只有0.3

所以，这种方法试用于数量较少的表。

折叠法

折叠法就是，把一个数，分成若干份（长度视样本而定），然后把这若干份再相加，得到key，注意到，哈希表的长度就是每一份的长度。此外这里使用 1000开始的数据去测试，因为用的n是4

int FoldHash(int key, int n) //n为散列表长度的位数
{
    int count =1;int num = 10;int resul t = 0;

    {
        if(key/num>0) count++;
        else{
            break;
        }

    }
    int group = count/n;
    for(int i=0;i<count;i+=group)
    {

        for(int j=i;j<i+group;j++)
        {
            int single_num = (key/pow(10,j));
            single_num = single_num%10;
            num += single_num*pow(10,j);
        }
        result+=num;
    }
    return result;
}

还好，用的十万作为测试集合。

总结

  • 除模取余法整体较好，适用于样本100左右的数据。
  • 平均取中法，总的来说不是很好，但是对于数字值较大的数据可以尝试这种方法
  • 随机乘数法，适中，但是算法耗时较久，不适合大容量的哈希表创建
  • 基数转化法，试用于容量较小的表。总体不是很好，而且计算复杂
  • 折叠法，目前为止，我觉得最好的方法，但是数据大小有下限，不能比他的长度还小，否则补0算法还得改。

哈希表

哈希表目前有两种形式的表，一种是开放地址法，一种是链式存储。下面用除模取余法分别建立两种表来看一下查找效率。

开放地址法

这种方法不是特别好。具体实现就是，如果出现冲突，就把这一个数往下一个（上一个）位置存储，若下一个位置已经有了，则继续往下一个。空间利用率高，但是，会让整个哈希表变得非常混乱。查找起来损耗的时间也多。

int openmap[1001];

void openadd()
{
    memset(openmap,-1,1001);
    for(int i = 0; i<len(list1); i++)
    {
        int j = modeKey(list1[i]);
        if(openmap[j] == -1)//还没有插入这个数
        {
            openmap[j] = list1[i];
        }
        else{
            int cc = j;
            while(openmap[cc+1] != -1) 
            { 
                if((cc+1)>1000)
                {
                    cc = 0;
                }
            cc++;
            }
            openmap[cc+1] = list1[i];
        }
    }
}

链表法

这种方法较好，如果遇到冲突的就再当前值后面加上，冲突值（把数据扩充为结构体）类似于bins的管理机制fasbin的实现。

首先建立结构体，和全局数组。

struct node{
    int keyy;
    node* next;
};
node* stor = new node[1001];

初始化哈希链表

void hashLinkMap()//链表法
{
    for(int i = 0; i<1001; i++)
    {
        node* tmp = new node;
        tmp->next = NULL;
        tmp->keyy = -1;//这个表示链表尾巴
        stor[i] = *tmp;
    }
    int ii = 0;
    while(ii<len(list1))
    {
        int j = list1[ii]%1001;
        if(stor[j].keyy != -1)
        {
            node* ptr = new node;
            ptr->keyy = list1[ii];
            ptr->next = NULL;

            node* tmp1;
            tmp1 = &stor[j];
            //cout<<tmp1->keyy << "and" << tmp1->next<<endl;
            while(tmp1->next != NULL)
            {
                tmp1 = tmp1->next;
            }
            tmp1->next = ptr;
            //delete(tmp1);
        }
        else
        {
            node* ptr = new node;
            ptr->next =NULL;
            ptr->keyy = list1[ii];
            stor[j] = *ptr;
            delete(ptr);
        }
        ii++;
    }
}

这个分号调了两个多小时，就是不给我报错，发现自己真傻。
